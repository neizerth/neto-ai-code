Кейс: Разработка модуля для обработки
заказов в интернет-магазине
Контекст:
Вы — старший разработчик в команде интернет-магазина. К вам
обратился младший коллега, который пытался автоматизировать свою
работу с помощью ИИ-ассистента, но столкнулся с проблемами. Его
задачей было — разработать и поддерживать модуль обработки заказов

обработки заказов он пробовал:
1. Сгенерировать новый код для обработки заказов.
2. Перевести часть логики с Python на Java.
3. Исправить ошибку в существующем коде.
4. Провести рефакторинг метода для улучшения читаемости.
Однако результаты были неутешительными: сгенерированный код не
интегрировался в проект, перевод между языками нарушал логику
расчетов, а попытки рефакторинга вносили новые баги. Коллега
разочарован и считает, что ИИ бесполезен для реальной разработки.
Ваша задача:

решили провести небольшой код-ревью его промптов. Для этого
