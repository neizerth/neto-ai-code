## Задание 2. Транскодинг (перевод кода между языками)

**Ситуация:**  
Есть короткая функция на JavaScript — нормализация номера телефона (оставить только цифры и лидирующий `+`). Её нужно перенести в Python 3 для использования в другом сервисе. Сохранить логику и обработку пустой строки.

**Исходный код (JavaScript, до 20 строк):**

```js
function normalizePhone(input) {
  if (typeof input !== 'string' || input.length === 0) return '';
  const trimmed = input.trim();
  const hasPlus = trimmed.startsWith('+');
  const digits = trimmed.replace(/\D/g, '');
  return (hasPlus ? '+' : '') + digits;
}
```

**Ваша задача:**

1. Составьте **структурированный промпт** для перевода в Python: исходный и целевый язык, сохранение поведения (пустая строка → `''`, обрезка пробелов, сохранение `+`, только цифры), идиоматичный Python (типы, стиль).
2. Укажите шаблон ответа (код функции + 1–2 предложения по отличиям).
3. (Опционально) Выполните промпт и проверьте на примерах `" +7 (999) 123-45-67 "`, `""`, `"abc"`.

## Задание 3. Фикс ошибки

**Ситуация:**  
Функция считает среднее арифметическое элементов массива. При пустом массиве происходит деление на ноль. Нужно исправить только эту функцию, не менять сигнатуру: при пустом массиве возвращать `0`.

**Исходный код (до 20 строк):**

```js
function average(numbers) {
  let sum = 0;
  for (let i = 0; i < numbers.length; i++) {
    sum += numbers[i];
  }
  return sum / numbers.length;
}
```

**Ваша задача:**

1. Составьте **промпт для фикса**: диагноз (деление на ноль при `numbers.length === 0`), границы (менять только тело функции), ожидаемое поведение (пустой массив → `0`).
2. Укажите шаблон ответа (только исправленная функция + краткий комментарий).
3. (Опционально) Выполните промпт и проверьте вызов `average([])`.

## Задание 4. Рефакторинг

**Ситуация:**  
Функция определяет, попадает ли возраст в одну из трёх категорий. Нужно улучшить читаемость (убрать «магические числа», упростить условия), **не меняя поведение**.

**Исходный код (до 20 строк):**

```js
function getAgeGroup(age) {
  if (age < 0) return 'invalid';
  if (age >= 0 && age <= 12) return 'child';
  if (age >= 13 && age <= 17) return 'teen';
  if (age >= 18) return 'adult';
  return 'invalid';
}
```

**Ваша задача:**

1. Составьте **промпт для рефакторинга**: контекст (JS), критерии (именованные границы возрастов, без дублирования, читаемость), **жёсткое ограничение** — результат для тех же входов не меняется.
2. Укажите шаблон ответа (рефакторированная функция + краткое пояснение).
3. (Опционально) Выполните промпт и проверьте граничные значения (0, 12, 13, 17, 18, отрицательное).

### 3. Фикс ошибки (JavaScript)

**Оптимальный промпт:**

**(Роль)** Ты — внимательный разработчик, следуешь принципу минимальных изменений при исправлении багов.

**(Диагноз)** Функция `average` падает или возвращает `NaN` при пустом массиве, так как выполняется деление на ноль: `sum / numbers.length` при `numbers.length === 0`.

**(Задача на фикс)** Измени только тело функции `average`, чтобы при пустом массиве возвращалось `0`. Сигнатуру не менять.

Исходный код:
```js
function average(numbers) {
  let sum = 0;
  for (let i = 0; i < numbers.length; i++) {
    sum += numbers[i];
  }
  return sum / numbers.length;
}
```

**(Границы и ограничения)**
1. Не менять сигнатуру функции.
2. Не менять поведение для непустого массива — формула среднего должна остаться той же.
3. Минимум правок: только добавить проверку на пустой массив.

**(Критерии успеха и ожидаемое поведение)**
- `average([])` возвращает `0`.
- Для любого непустого массива чисел результат совпадает с исходной реализацией.

**(Шаблон ответа)** Предоставь только исправленную функцию и краткий комментарий, где и зачем добавлена проверка.
