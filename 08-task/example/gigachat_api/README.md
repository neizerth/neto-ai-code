## Пример: GigaChat API + Streamlit

Пример: пользователь пишет блюдо → приложение возвращает список покупок.

### Что нужно

- Python 3.10+
- Доступ к GigaChat API (ключ авторизации)
- Сертификаты НУЦ Минцифры — см. документацию GigaChat (часто без них не работает)

### Сертификаты НУЦ Минцифры

Если сертификатов нет, часто бывает ошибка вида:  
`[SSL: CERTIFICATE_VERIFY_FAILED] certificate verify failed ...`

Есть два варианта.

**Вариант 1 (быстро, на уровне Python/приложения).**

```bash
curl -k "https://gu-st.ru/content/lending/russian_trusted_root_ca_pem.crt" -w "\n" >> $(python3 -m certifi)
```

Если вы используете `gigachat`, можно также скачать файл сертификата и передать путь через `GIGACHAT_CA_BUNDLE_FILE` (см. `.env.example`).

**Вариант 2 (на уровне ОС).**

- macOS/Windows: скачайте сертификат на портале Госуслуг и установите по инструкции.
- Linux: скачайте корневой и выпускающий сертификаты в `.crt`, установите в систему и обновите хранилище сертификатов.

### Быстрый запуск

1) Создайте и активируйте окружение:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

2) Поставьте зависимости:

```bash
pip install -r requirements.txt
```

3) Настройте переменные окружения:

```bash
cp .env.example .env
```

Откройте `.env` и вставьте `GIGACHAT_CREDENTIALS=...`.

Если в вашем окружении нужны сертификаты, установите их (см. документацию GigaChat).  
Для демо можно отключить проверку сертификатов: `GIGACHAT_VERIFY_SSL_CERTS=0`.

4) Запуск UI:

```bash
streamlit run streamlit_app.py
```

### Проверки (для самопроверки)

- **пустой ввод**: нажмите «Сгенерировать» с пустым полем
- **слишком длинный ввод**: вставьте очень длинный текст
- **ошибка доступа**: временно очистите `GIGACHAT_CREDENTIALS` и запустите снова

### Безопасность

- не пишите ключи в коде
- храните ключи в переменных окружения/`.env` (и не коммитьте `.env`)
